任意命令执行

当应用需要调用一些外部程序去处理内容的情况下，就会用到一些执行系统命令的函数。当用户可以控制命令执行函数中的参数时，就可以注入恶意系统命令到正常命令中，造成命令执行漏洞。

脚本语言优点是简洁、方便，但也伴随着一些问题，如速度慢、无法接触系统底层，开发人员需要一些除去web的特殊功能时，就要调用一些外部程序。

应用在调用这些函数执行系统命令的时候，如果将用户的输入作为系统命令的参数拼接到命令行中，又没有对用户输入的进行过滤的情况下，就会造成命令执行漏洞。

PHP中常见可以调用外部程序的函数：
system exec shell_exec passthru popen proc_popen

危害
继承web服务程序的权限，去执行系统命令
读写文件
反弹shell
控制整个网站 甚至控制整个服务器 进一步内网渗透 等

典型漏洞代码：
<?php
system($_GET[‘cmd’])
?>
system执行get得到的任意参数

过滤绕过：
; & |

linux下的sendmail命令自带读写文件功能，可以用了些webshell

linux shell环境下双引号中间的变量也是可以被解析的，我们可以在双引号内利用反引号执行任意命令

system(“/bin/prog –p=’$arg’”);
这种情况下我们要下闭合单引号才能执行命令：system(“/bin/prog -p=’aaa’|id”)

修复：
能使用脚本解决的工作，不要调用其他程序处理。尽量少使用执行命令的函数，并在disable_functions中禁用。
/etc/php.ini文件中：
查找到 disable_functions ，添加需禁用的函数名，如下：
phpinfo,eval,passthru,exec,system,chroot,scandir,chgrp,chown,shell_exec,proc_open,proc_get_status,ini_alter,ini_alter,ini_restore,dl,pfsockopen,openlog,syslog,readlink,symlink,popepassthru,stream_socket_server,fsocket,fsockopen

对于可控点事程序参数的情况，使用escapeshellcmd函数进行过滤(出去字符串中的特殊符号)

参数的值尽量使用引号包裹，并在拼接前调用addslashes进行转义。

利用的前提：
代码中存在调用系统命令的函数
函数中存在我们可控的点
这个点没有经过过滤。

任意代码执行

很多CMS为了设置的灵活性，都会选择使用eval函数来处理内容，但是处理的同时没有检查用户是否可以控制被处理的字符串。
PHP中能造成代码注入的主要函数：
eval() assert()
callback函数：preg_replace + /e 模式
反序列化函数：unserialize()
python中的exec()
危害 执行任意的代码，写入webshell 甚至控制服务器

示意代码：
源代码：
<?php
$data = $_GET[‘data’];
eval(“\$ret = $data;”);
echo $ret;

payload:
1;phpinfo()
phpinfo()
${phpinfo()}
分析
eval(“\$ret = $data;”);
eval(“\$ret = 1;phpinfo();”)
eval: 定义了一个变量$ret=1;执行了一段代码：phpinfo();

源代码：
<?php
$data = $_GET[‘data’];
echo “\$ret = ‘$data’;”;
eval(“\$ret = strtolower(‘$data’);”);
echo $ret;

payload：
‘);phpinfo();//

分析
eval(“\$ret = strtolower(‘$data’);”);
eval(“\$ret = strtolower(‘’);phpinfo();//‘);”);
eval:定义了一个变量$ret=strtolower(‘’);执行一段代码phpinfo();//注释了’)

源代码：
<?php
$data = $_GET[‘data’];

#echo “\$ret = strtolower(\”$data\”);”;
eval(“\$ret = strtolower(\”$data\”);”);
echo $ret;

/ strtolower() 函数把字符串转换为小写。注释：该函数是二进制安全的。 /
/ {$a}，由于这里a是变量，函数必须{}括起来才能引用返回值， 在双引号包围的情况下 实际上就是，{${函数}} /

payload:
① {${phpinfo()}}
② “);phpinfo();//

分析①
eval(“\$ret = strtolower(\”$data\”);”);
eval(“\$ret = strtolower(\”{${phpinfo()}}\”);”);
eval定义了一个变量$ret=”{${phpinfo()}}”;

源代码：
<?php
$data = $_GET['data'];
echo $data;
preg_replace('/<data>(.*)<\/data>/e', '$ret = "\\1";', $data);
echo $ret; 

payload:
    <data>{${phpinfo()}}</data>
分析：
       preg_replace('/<data>(.*)<\/data>/e', '$ret = "\\1";', $data);   //正则匹配出（.*）传递给$ ret则：
    $ret="{${phpinfo()}}"
    在php中""里的内容用{}括起来会理解为代码执行

修复方案：
能使用json保存数组、对象就使用json，不要将php对象保存成字符串，否则读取的时候需要使用eval
对于必须使用eval的情况，一定要保证用户不能轻易接触eval的参数(用正则严格判断输入的数据格式)
对于字符串，一定要使用单引号包裹可控代码，并在插入前进巷addslashes
addslashes()函数返回在预定义字符之前添加反斜杠的字符串。
预定义字符是：
单引号（’）
双引号（”）
反斜杠（\）
NULL

放弃使用preg_replace的e修饰符
使用preg_replace_callback()替换
如果使用preg_replace的e修饰符，保证第二个参数中，对于正则匹配出的对象，用单引号包裹。

Read_nmap

导语：

nmap的英文全称是”Network Mapper”,nmap可以用来快速地扫描大型网络或单个主机。nmap主要有四个方面的扫描功能：主机发现、端口扫描、应用与版本侦测、操作系统侦测


主机发现：

nmap -sn 192.168.1.100-200 //扫描局域网192.168.1.100-192.168.1.200范围内活动的IP主机
nmap -v //显示过程
nmap -A //进行完整全面的扫描：主机发现、端口扫描、应用与版本侦测、操作系统侦测
nmap -n/-R //不进行DNS解析/总是解析
nmap -iL –指定ip.txt

端口扫描：

  • nmap -sS #使用TCP SYN方式对目标主机进行扫描TCP端口
  • nmap -sU #使用UDP扫描方式确定主机UDP端口
  • nmap -F #快速扫描TOP100的端口
  • nmap -Pn #跳过主机发现进行端口扫描
  • nmap -p 3389 #扫描指定的3389端口是否开放
  • nmap –traceroute #追踪路由节点
  • nmap –top-ports 100 #扫描开放概率最高的前100个端口
例：nmap -sS -sU --top-ports 100 192168.136.86

版本侦测：

  • nmap -sV #进行版本侦测
  • nmap –version-all #尝试使用所有的probes进行侦测 （intensity 9)
  • nmap –version-trace #显示详细的侦测过程信息

操作系统侦测：

  • nmap -O #进行操作系统侦测


nmap进阶使用：

防火墙/IDS规避
规避原理：1、分片；2、IP诱骗；3、IP伪装；4、指定源端口；5、扫描延时；
  • -f #使用分片？？？？不太懂。。。
  • -D<192.168.1.1,192,168,1,2,me> #IP诱骗 ME为自己的IP
  • -S<192.168.33.55> #IP伪装 伪装成其他IP
  • -g 3355 #指定源端口 表示自己的端口使用3355
  • -e echo #使用echo网卡发送数据包 
例：nmap -v -F -Pn -D 192.168.1.2,192.168.4.6,ME -g 3355 127.0.0.1
    显示过程信息，-F快速扫描top100的端口，-D使用ip诱骗掩饰自己真实IP（ME为自己IP），-g 3355使用自己的3355端口 127.0.0.1为扫描的目标IP.

数据库渗透测试：

  • nmap -p 3306 –script mysql-databases –script-args mysqluser=root,mysqlpass 127.0.0.1

列举目标数据库：

  • nmap -p 3306 –script=mysql-empty-password 123.206.9.45 #检查目标mysql密码

信息收集：

  • nmap –script ip-geolocation-* www.baidu.com #IP地址收集 收集www.baidu.com域名包含的IP信息

  • nmap –script hostmap-ip2hosts baidu.com #IP反查 可以将所有绑定在该IP上的域名显示出来

  • nmap –script dns-brute baidu.com #DNS信息收集

  • nmap –script smb-check-vulns.nse -p 445 127.0.0.1 #对SMB漏洞进行扫描

  • nmap –script external baidu.com #Whois查询
